# 구독자 추가하기

## 이 글에서는 <a href="#h_01hfnb10kyswn1hhz9spmnnpps" id="h_01hfnb10kyswn1hhz9spmnnpps"></a>

스티비 주소록에 구독자를 추가하는 세 가지 방법을 안내합니다. 구독자를 직접 입력해 한 명씩 추가하거나, CSV 파일로 한 번에 업로드하거나, 주소록 API로 외부 DB와 연동할 수 있습니다. 이 방법에 관해 알아봅니다.

***

## 구독자를 직접 추가하기 <a href="#h_01gfama1vb0pgg5a5gst8a7ys1" id="h_01gfama1vb0pgg5a5gst8a7ys1"></a>

구독자를 직접 추가하는 방법은 '주소록 목록'에서 추가하는 방법과 '구독자 목록'에서 추가하는 방법 두 가지가 있습니다.

### 주소록 목록에서 추가하기 <a href="#h_01gftbsyt4jd50zgf3qdyfwdgb" id="h_01gftbsyt4jd50zgf3qdyfwdgb"></a>

1. \[주소록]을 클릭해 주소록 목록으로 이동합니다.
2. 구독자를 추가하고 싶은 주소록 오른쪽의 \[추가하기(∨)]를 누릅니다.
3. \[직접 추가하기]를 선택합니다.
4. 구독자 정보를 입력한 뒤 \[추가하기]를 누르면 구독자가 추가됩니다.

![](<../../.gitbook/assets/1 (6).png>)

### 구독자 목록에서 추가하기 <a href="#h_01gftbt53qcp92bsc0qkxrydx8" id="h_01gftbt53qcp92bsc0qkxrydx8"></a>

1. \[주소록]을 클릭해 구독자를 추가하고 싶은 주소록의 \[구독자 목록] 화면으로 이동합니다.
2. 오른쪽 위 \[구독자 추가하기 → 직접 추가하기]를 선택합니다.
3. 구독자 정보를 입력한 뒤 \[추가하기]를 누르면 구독자가 추가됩니다.

![](<../../.gitbook/assets/2 (4).png>)



### 구독자 정보 입력하기 <a href="#h_01gftbtdnvb0z4g8117m5anzqe" id="h_01gftbtdnvb0z4g8117m5anzqe"></a>

\[직접 추가하기]를 누르면 구독자 정보를 입력하는 화면으로 이동합니다. 정보를 입력한 뒤 \[추가하기]를 누르면 구독자가 추가됩니다.

한 명 이상의 구독자를 등록하려면 입력 폼 아래 \[+ 추가하기]를 클릭해 폼을 추가하면 됩니다. 추가하는 구독자 수에 따라 처리 시간이 달라질 수 있습니다. 구독자를 추가하는 도중 화면을 떠나도 완료될 때까지 자동으로 진행됩니다.

#### **이메일 주소를 포함한 구독자 정보**

이메일 주소, 이름, 광고성 정보 수신 동의 여부 등 구독자 정보를 입력합니다. 이메일 주소는 필수 값입니다. 광고성 정보 수신 동의 여부는 '동의함' 또는 '동의하지 않음'으로 선택할 수 있습니다.

_\* 입력해야 하는 구독자 정보의 종류는_ [_사용자 정의 필드_](understanding-subscriber-info.md#h_01gw45zrwcjd1eg1cam4vhw25m)_에서 관리할 수 있습니다._

#### **추가하는 필드에 값이 없으면 기존 값을 삭제하고 빈 값으로 업데이트합니다.**

이미 주소록에 등록된 구독자를 다시 추가할 때, 기존에 등록된 데이터를 어떻게 처리할지 결정할 수 있는 옵션입니다.

이 옵션을 체크하면, 새로 추가하는 구독자 정보 중 특정 값(예: 이름)이 비어 있는 경우 기존에 저장된 값은 사라지고 빈 값으로 업데이트됩니다. 구독자의 정보가 중복되는 정보인지 여부는 이메일 주소를 기준으로 판단합니다.

예를 들어, dooly@stibee.com 구독자의 이름을 '둘리'로 등록한 뒤, 이름을 비워 두고 같은 이메일 주소로 다시 추가하면 이름이 빈 값으로 업데이트됩니다. 이 옵션을 체크하지 않으면 기존 이름인 '둘리'가 유지됩니다.

#### **그룹을 선택하여 구독자를 추가하시겠습니까?**

구독자를 추가할 때 특정 그룹에 할당할지 여부를 설정하는 옵션입니다. '네, 그룹을 선택해 구독자를 추가할게요'를 선택하면 특정 그룹에 할당되도록 설정할 수 있습니다.

_\* 그룹 기능은_ [_그룹 사용하기_](../classify-subscribers/how-to-use-groups.md) _도움말을 참고해 주세요._

![](<../../.gitbook/assets/4 (3).png>)



## 구독자 파일로 추가하기

기존에 보유한 구독자 정보가 있거나 한 번에 많은 구독자를 추가해야 할 때는 \[파일로 추가하기] 기능을 사용합니다. 구독자 정보를 CSV 파일로 만들어 업로드할 수 있습니다.

파일로 구독자를 추가하는 방법은 '주소록 목록'에서 추가하는 방법과 '구독자 목록'에서 추가하는 방법 두 가지가 있습니다.

{% hint style="info" %}
파일 업로드가 되지 않는다면 [구독자 파일 업로드가 안 될 때](../../trouble-shooting/subscriber-upload-issues.md) 도움말을 참고해 문제를 해결해 보세요. CSV 파일 형태로 만들기 어렵다면 [샘플 파일](https://stibee.com/download/%EC%8A%A4%ED%8B%B0%EB%B9%84_%EC%A3%BC%EC%86%8C%EB%A1%9D_%EC%83%98%ED%94%8C.csv)을 내려받은 뒤 수정해서 사용해 주세요.
{% endhint %}

### 주소록 목록에서 파일로 추가하기 <a href="#h_01gftbtwx6ky33ch5gz5ne3n01" id="h_01gftbtwx6ky33ch5gz5ne3n01"></a>

1. \[주소록]을 클릭해 주소록 목록으로 이동합니다.
2. 구독자를 추가하고 싶은 주소록 오른쪽의 \[추가하기(∨)]를 누릅니다.
3. \[파일로 추가하기]를 선택합니다.
4. 파일을 업로드한 뒤 \[추가하기]를 누르면 구독자가 추가됩니다.

<figure><img src="../../.gitbook/assets/5 (4).png" alt=""><figcaption></figcaption></figure>

### 구독자 목록에서 파일로 추가하기 <a href="#h_01gftbv72ptg4apx4xskgnm4jk" id="h_01gftbv72ptg4apx4xskgnm4jk"></a>

1. \[주소록]을 클릭해 구독자를 추가하고 싶은 주소록의 \[구독자 목록] 화면으로 이동합니다.
2. 오른쪽 위 \[구독자 추가하기 → 파일로 추가하기]를 선택합니다.
3. 파일을 업로드한 뒤 \[추가하기]를 누르면 구독자가 추가됩니다.

![](<../../.gitbook/assets/6 (4).png>)

### 구독자 정보 설정하기

파일을 업로드한 뒤 \[다음]을 누르면 구독자 정보를 설정하는 화면으로 이동합니다. 설정을 완료한 뒤 \[추가하기]를 누르면 업로드가 시작됩니다. 추가하는 구독자 양에 따라 완료 시간이 달라질 수 있습니다. 구독자를 추가하는 도중 화면을 떠나도 완료될 때까지 자동으로 진행됩니다.

#### **구독자 정보와 사용자 정의 필드 연결하기**

업로드한 구독자 정보를 어떤 사용자 정의 필드와 연결할지 선택합니다. 이메일 주소를 제외한 나머지 필드는 '사용 안 함'으로 자동 선택되어 있습니다. 업로드해야 하는 정보가 있다면 해당하는 사용자 정의 필드를 연결해 주세요.

파일에 등록된 정보와 일치하는 사용자 정의 필드가 없다면, 이 단계에서 \[+ 사용자 정의 필드 추가]를 눌러 바로 생성할 수 있습니다.

<figure><img src="../../.gitbook/assets/구독자 추가하기.png" alt=""><figcaption></figcaption></figure>

#### 업로드 한 파일의 첫 번째 행을 제외하고 구독자를 추가합니다.

업로드한 파일의 첫 번째 행을 구독자로 추가하지 않는 옵션입니다.

파일을 업로드하면 첫 번째 행도 구독자 데이터로 불러옵니다. 첫 번째 행에 열 이름(헤더)이 입력되어 있다면 이 옵션을 체크하세요. 체크하면 첫 번째 행은 구독자로 추가되지 않습니다. 첫 번째 행부터 실제 구독자 데이터가 시작된다면 체크하지 않아도 됩니다.

#### **추가하는 필드에 값이 없으면 기존 값을 삭제하고 빈 값으로 업데이트합니다.**

이미 주소록에 등록된 구독자를 다시 추가할 때, 기존에 등록된 데이터를 어떻게 처리할지 결정할 수 있는 옵션입니다.

이 옵션을 체크하면, 새로 추가하는 구독자 정보 중 특정 값(예: 이름)이 비어 있는 경우 기존에 저장된 값은 사라지고 빈 값으로 업데이트됩니다. 구독자의 정보가 중복되는 정보인지 여부는 이메일 주소를 기준으로 판단합니다.

예를 들어, dooly@stibee.com 구독자의 이름을 '둘리'로 등록한 뒤, 이름을 비워 두고 같은 이메일 주소로 다시 추가하면 이름이 빈 값으로 업데이트됩니다. 이 옵션을 체크하지 않으면 기존 이름인 '둘리'가 유지됩니다.

#### **그룹을 선택해 구독자를 추가하시겠습니까?**

구독자를 추가할 때 특정 그룹에 할당될지 여부를 설정할 수 있는 옵션입니다.

'네, 그룹을 선택해 구독자를 추가할게요'를 선택하면 구독자를 추가할 때 특정 그룹에 할당되도록 설정할 수 있습니다.

_\* 그룹 기능은_ [_그룹 사용하기_](../classify-subscribers/how-to-use-groups.md) _도움말을 참고해 주세요._



### 파일로 추가할 때 수신거부 상태로 추가하기 <a href="#h_01gfam9r8typybhwpebnjf382c" id="h_01gfam9r8typybhwpebnjf382c"></a>

수신거부 상태의 구독자를 파일로 추가하거나, 구독자를 추가하면서 수신거부 상태로 등록할 수 있습니다.

1. CSV 파일에서 수신거부 여부를 표시할 열을 추가하고, 수신거부 상태로 추가할 구독자는 이 열에 Y를 입력합니다.
2. 구독자 정보 설정 화면에서 해당 열을 \[이메일 수신거부] 필드와 매칭합니다.

수신거부 열에 Y가 입력된 구독자는 수신거부 상태로 추가됩니다. 이미 등록된 구독자라면 구독 상태가 수신거부로 변경됩니다.

### 파일로 추가할 때 광고성 정보 수신 동의 설정하기

파일로 구독자를 추가할 때, 광고성 정보 수신 동의 여부를 함께 설정할 수 있습니다.

1. CSV 파일에서 광고성 정보 수신 동의 여부를 표시할 열을 추가하고, 동의한 구독자는 이 열에 Y를 입력합니다.
2. 구독자 정보 설정 화면에서 해당 열을 \[광고성 정보 수신 동의] 필드와 매칭합니다.

광고성 정보 수신 동의 열에 Y가 입력된 구독자는 동의 상태로 추가됩니다. N 또는 빈칸인 구독자는 '동의하지 않음(빈칸)'으로 표시됩니다.

## 구독자 추가 결과 확인하기 <a href="#results-of-adding" id="results-of-adding"></a>

구독자 추가가 완료되면 결과가 화면에 표시됩니다. '파란 글씨(n명)'를 클릭하면 해당 결과의 이메일 주소를 확인할 수 있습니다. \[파일로 내보내기]를 누르면 목록을 내려받을 수 있는 링크가 담긴 이메일이 발송됩니다.

### **상태 외 정보 업데이트**

추가한 구독자 중 이미 주소록에 존재하는 구독자가 있어 이메일 주소 외의 정보가 업데이트됐습니다. 이메일 주소가 같은데 이름 필드 값이 다르면, 마지막에 등록한 파일을 기준으로 구독자 정보가 업데이트됩니다.

### **이메일 주소 없음**

업로드한 파일에 특정 구독자의 이메일 주소가 없습니다. 원본 파일을 확인해 주세요.

### **이메일 주소 형식 오류**

이메일 주소가 입력됐지만 형식이 올바르지 않습니다. 이메일 주소는 example@stibee.com과 같은 형식이어야 합니다. 원본 파일을 확인해 주세요.

### **이메일 주소 중복 요청**

같은 이메일 주소를 가진 구독자가 여러 명 있어 한 명만 추가됐습니다. 가장 처음에 요청된 이메일 주소를 기준으로 구독자 정보가 업데이트됩니다.

<figure><img src="../../.gitbook/assets/10 (2).png" alt=""><figcaption></figcaption></figure>



## DB 연동하기 <a href="#h_01gfam9r8typybhwpebnjf382c" id="h_01gfam9r8typybhwpebnjf382c"></a>

[주소록 API](../../api-webhook/overview.md#api)를 사용하면 홈페이지나 서비스의 회원 DB와 스티비 주소록을 연동할 수 있습니다. DB에 새 회원 정보가 등록되거나 기존 정보가 변경될 때, 스티비 주소록에 자동으로 반영되도록 설정할 수 있습니다.

반대로 [주소록 웹훅](../../api-webhook/list-webhook.md)을 사용하면 스티비 주소록의 구독자 정보가 변경될 때 사용자 서버로 알림을 받을 수 있습니다.

주소록 API와 웹훅을 모두 사용하면 DB와 주소록을 완전히 동기화해 운영할 수 있습니다.



## 구독 받기

보유한 구독자 정보가 없다면, 구독 신청을 통해 처음부터 구독자를 모집할 수 있습니다.

스티비에서는 주소록마다 연결된 구독 신청 페이지를 제공합니다. 이를 \[구독 폼]이라고 부릅니다. 기본 제공 페이지 그대로 활용하거나, 이메일 아카이빙 기능과 결합된 페이지 형태로 운영할 수도 있습니다. 구독 폼 코드를 직접 홈페이지나 블로그에 설치할 수도 있습니다.
