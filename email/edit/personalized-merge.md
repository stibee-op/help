# 개인화된 메시지 보내기

## 이 글에서는

이 글에서는 메일머지 기능으로 구독자 이름, 회사명 등 구독자별로 다른 정보를 담아 개인화된 이메일을 보내는 방법을 안내합니다.

***

## 메일머지 이해하기 <a href="#h_01gg72mtzptjnc7h911r1ss40n" id="h_01gg72mtzptjnc7h911r1ss40n"></a>

메일머지는 여러 명의 구독자에게 이메일을 보낼 때 각 구독자에게 맞는 개인화된 정보를 자동으로 포함하는 기능입니다. 이메일 제목이나 본문에 구독자 이름, 회사명, 가입일 등을 추가해서 발송할 수 있습니다.

메일머지를 사용하면 실제 발송 시 각 구독자의 정보가 자동으로 치환됩니다. 같은 내용의 이메일을 보내더라도 구독자 개개인에게 맞춘 느낌을 줄 수 있습니다.

<figure><img src="../../.gitbook/assets/개인화된 메시지 (1).png" alt=""><figcaption></figcaption></figure>



## 메일머지 사용하기

💬 이 내용은 **스탠다드, 프로, 엔터프라이즈 요금제**에 해당하는 도움말입니다.

메일머지는 주소록의 \[사용자 정의 필드]에 저장된 정보를 이메일 발송 시 자동으로 불러옵니다. 어떤 정보를 불러올지는 사용자 정의 필드에 설정한 키(key) 값이 기준입니다.

메일머지를 사용하기 전에 아래 내용을 준비해 주세요.

1. 구독자별로 다른 정보를 보내고 싶은 사용자 정의 필드를 등록합니다.
2. 사용자 정의 필드에 구독자별로 정보를 추가합니다.
3. 사용자 정의 필드의 키값을 확인합니다.

사용자 정의 필드를 만드는 방법은 [구독자 정보 이해하기](https://help.stibee.com/list/adding-managing-subscriber/understanding-subscriber-info) 도움말을 참고해 주세요.



### 메일머지 형식 이해하기

메일머지 치환자는 '$', '%', 사용자 정의 필드의 '키값' 세 가지 요소로 구성됩니다. 아래 형식으로 입력합니다.

```
$%사용자 정의 필드 키값%$
```

예를 들어 A 주소록에 '회사명'이라는 사용자 정의 필드를 추가하고 키값을 company로 설정했다면, 이메일 제목이나 본문에 아래와 같이 입력합니다.

```
$%company%$
```

A 주소록에서 dooly@stibee.com 구독자의 '회사명' 필드 값이 '스티비'로 등록되어 있다면, 발송 시 아래와 같이 자동으로 변환됩니다.

```
입력: $%company%$ 담당자님 안녕하세요.
발송: 스티비 담당자님 안녕하세요.
```

{% hint style="info" %}
정보가 저장되지 않은 구독자에게 표시할 기본값을 설정할 수도 있습니다. 예를 들어 \[이름] 필드에 정보가 없는 구독자에게는 '고객'으로 표시되도록 할 수 있습니다. 메일머지 기본값은 \[사용자 정의 필드] 설정에서 입력합니다. 자세한 내용은 [구독자 정보 이해하기](https://help.stibee.com/list/adding-managing-subscriber/understanding-subscriber-info) 도움말을 참고해 주세요.
{% endhint %}



### 메일머지 치환자 입력하기

메일머지는 입력 버튼을 사용하거나, 직접 입력하거나, HTML 코드에 넣는 세 가지 방법으로 입력할 수 있습니다.

#### 메일머지 입력 버튼으로 입력하기 <a href="#h_01gg72nht9e5hwjss9zkz6vwf7" id="h_01gg72nht9e5hwjss9zkz6vwf7"></a>

이메일 제목 입력창과 편집기의 텍스트 상자를 사용하는 경우, 툴바 윗부분에 메일머지 치환자를 입력할 수 있는 버튼이 표시됩니다. 버튼을 클릭해 입력하고 싶은 \[사용자 정의 필드]를 선택하면 메일머지 형식에 맞는 값이 자동으로 입력됩니다.

<figure><img src="../../.gitbook/assets/개인화된 메시지2.png" alt=""><figcaption></figcaption></figure>

이메일 본문에서도 같은 방법으로 메일머지 치환자를 입력할 수 있습니다.

<figure><img src="../../.gitbook/assets/개인화된 메시지3.png" alt=""><figcaption></figcaption></figure>

#### 직접 입력하기 <a href="#h_01gg72npydwncy0b92apvnhd39" id="h_01gg72npydwncy0b92apvnhd39"></a>

버튼을 사용하지 않고 형식에 맞춰 메일머지 치환자를 직접 입력해도 됩니다.

```
$%name%$
```



### HTML 코드에 입력하기

\[HTML 템플릿으로 만들기] 또는 에디터의 \[HTML 상자]에서도 메일머지를 사용할 수 있습니다. 메일머지 치환자를 넣고 싶은 위치에 형식에 맞춰 직접 입력하면 됩니다. 예를 들어 HTML 코드 중간에 구독자 이름을 넣고 싶다면 아래와 같이 입력합니다.

{% code overflow="wrap" %}
```html
<div>
    안녕하세요. </br>
    $%name%$님. 구독을 신청해 주셔서 감사합니다.
</div>
```
{% endcode %}



### 메일머지 동작 확인하기

실제 발송하지 않아도 테스트 발송으로 메일머지 치환자가 잘 동작하는지 확인할 수 있습니다.

1. 이메일 편집을 마쳤다면 화면 오른쪽 위에 있는 \[테스트 발송하기]를 클릭합니다.
2. 테스트 이메일을 받을 이메일 주소를 입력합니다.
3. 이메일 주소 오른쪽에 표시되는 필드 정보에 원하는 값을 입력합니다.
4. 테스트 이메일을 보냅니다.

<figure><img src="../../.gitbook/assets/개인화된 메시지4.png" alt=""><figcaption></figcaption></figure>



## 메일머지 기본 필드 활용하기

사용자 정의 필드 외에도 스티비가 기본으로 제공하는 기본 필드에 메일머지를 적용할 수 있습니다. 기본 필드 종류는 사용할 수 있는 기본 필드 목록 도움말을 참고해 주세요.

{% hint style="info" %}
메일머지는 유료 요금제에서 제공되는 기능이지만, 기본 필드를 활용한 메일머지는 무료인 스타터 요금제에서도 사용할 수 있습니다.
{% endhint %}

메일머지 기본 필드만 활용해도 아래 시나리오를 간단하게 구현할 수 있습니다.

* 뉴스레터를 받아보고 있는 구독자 수를 자동으로 업데이트할 수 있습니다.
* 구독자가 직접 정보를 변경할 수 있는 화면으로 연결되는 URL을 간단하게 넣을 수 있습니다.
* 가장 최근에 발행한 뉴스레터 URL을 자동으로 업데이트할 수 있습니다.
* 누가 어떤 이메일을 통해 유입됐는지 성과를 구체적으로 추적할 수 있습니다.



### 이메일에 구독자 수 표시하기 <a href="#h_01gg9m6zytq2r7q4d46rhvzd3s" id="h_01gg9m6zytq2r7q4d46rhvzd3s"></a>

뉴스레터 구독자 수를 구독자에게 알려주고 싶다면 기본 필드 메일머지를 활용하면 됩니다. 구독자 수가 변경될 때마다 직접 수정할 필요 없이, 발송 시점의 구독자 수로 자동 치환됩니다.

1. 이메일에 \[텍스트 상자]를 추가합니다.
2. 문구를 작성하고, 구독자 수가 들어가야 하는 부분에 $%subscribers\_count%$ 를 입력합니다.&#x20;
   1. 예. 현재 $%subscribers\_count%$ 명이 뉴스레터를 구독하고 있어요.
3. 이메일을 발송합니다.

발송 시점에 주소록에 등록된 구독자가 1,500명이라면 아래와 같이 자동으로 변환됩니다.

```
작성 시: 현재 $%subscribers_count%$명이 뉴스레터를 구독하고 있어요.
발송 시: 현재 1,500명이 뉴스레터를 구독하고 있어요.
```

### 최근에 발행한 이메일의 공유용 URL 삽입하기 <a href="#h_bb4996600e" id="h_bb4996600e"></a>

기본 필드 메일머지를 활용하면, 같은 주소록에서 가장 최근에 보낸 이메일의 공유용 URL이 자동으로 입력되도록 할 수 있습니다.

1. 이메일 본문에 \[버튼 상자]를 삽입하거나 텍스트를 입력합니다.
2. 입력한 버튼 또는 텍스트 상자를 클릭하면 윗부분에 툴바가 표시됩니다. 툴바의 링크 버튼을 클릭하고, 오른쪽에 있는 메일머지 입력 버튼 \[{ }]을 클릭하면 사용할 수 있는 기본 필드가 표시됩니다.
3. 목록에서 \[가장 최근에 발송한 이메일의 공유용 URL]을 선택합니다.
4. 또는 메일머지 치환자 %permalink\_latest% 를 링크 입력창에 직접 입력해도 됩니다.

### 구독 정보 변경 URL 보내기 <a href="#h_01gg9m7a5e486y82nfwnadntkp" id="h_01gg9m7a5e486y82nfwnadntkp"></a>

기본 필드 메일머지를 사용하면 구독자가 자신의 구독 정보를 직접 변경할 수 있는 링크를 간단하게 넣을 수 있습니다.

1. 이메일에 \[버튼 상자]나 \[텍스트 상자]를 삽입합니다.
2. 삽입한 상자를 클릭하면 표시되는 툴바에서 링크 버튼을 클릭합니다.
3. 입력창 오른쪽에 있는 \[{ }]를 클릭하고 \[구독 정보 변경 화면 URL]을 선택합니다. 또는 %preferences% 를 링크에 직접 입력해도 됩니다.
4. 이메일을 발송하면 %preferences% 가 이메일이 발송된 주소록의 \[구독 정보 변경 화면] URL로 자동 변환됩니다.

### 이메일을 통한 서비스 유입 성과 측정하기 <a href="#h_01gg9m7fqz25c1h40fdde2qjye" id="h_01gg9m7fqz25c1h40fdde2qjye"></a>

구글 애널리틱스(Google Analytics), 앰플리튜드(Amplitude) 등을 활용하면 내 홈페이지로 유입된 사용자를 추적할 수 있습니다. 이때 UTM 파라미터(UTM parameter)를 사용하면 사용자가 어떤 채널, 어떤 마케팅 캠페인을 통해 유입됐는지도 기록할 수 있습니다.

메일머지 기본 필드 중 \[이메일 id] 필드를 UTM 파라미터와 함께 사용하면, 방문자가 구체적으로 어떤 이메일을 통해 유입됐는지 추적할 수 있습니다. 이메일은 6자리 숫자로 구성된 고유한 ID 값을 가지며, UTM 파라미터 링크에 \[이메일 id] 필드의 메일머지 치환자를 넣으면 발송 시 자동으로 이메일 ID로 변환됩니다.

아래 형식으로 URL을 구성해 이메일의 텍스트나 버튼에 넣으면, 이메일별 유입 성과를 더 자세하게 측정할 수 있습니다.

*   예: [https://stibee.com/?utm\_medium=email\&utm\_campaign=$%email\_id%$](https://stibee.com/?utm_medium=email\&utm_campaign=$%email_id%$)

    _\* 실제 사용 시에는 위 예시를 참고해 URL을 수정해서 사용해 주세요_



## 키값이 중복될 때 동작 이해하기

\{% hint style="warning" %\} \*주의: 사용자가 직접 설정한 '사용자 정의 필드'의 키(key) 값과 스티비에서 기본으로 제공하는 '기본 필드'의 키(key) 값이 중복되면, 사용자 정의 필드가 우선 적용됩니다. \{% endhint %\}

예를 들어 아래와 같이 사용자 정의 필드를 설정한 경우를 살펴보겠습니다.

* 사용자 정의 필드 이름: 구독자 이름
* 설정한 키(key) 값: list\_name

list\_name은 기본 필드에 이미 존재하는 키값입니다. 이 키값으로 사용자 정의 필드를 생성하면, 기본 필드 메일머지 대신 직접 생성한 사용자 정의 필드 기준으로 동작합니다.

* **기존**(사용자 정의 필드에 구독자 이름(list\_name)을 추가하기 전)
  * 발송 전: $%list\_name%$ 주소록에 추가되었습니다.
  * 발송 후: 기본 주소록에 추가되었습니다.
* **변경**(사용자 정의 필드에 구독자 이름(list\_name)을 추가한 뒤)
  * 발송 전: 안녕하세요, $%list\_name%$님!
  * 발송 후: 안녕하세요, 둘리님!

사용자 정의 필드를 설정할 때는 기본 필드의 키값과 중복되지 않는지 확인하는 것을 추천합니다. 기본 필드 종류는 사용할 수 있는 [기본 필드 목록](https://help.stibee.com/list/adding-managing-subscriber/understanding-subscriber-info#default-fields) 도움말을 참고해 주세요.
